Ressource Group -> AppService Plan -> AppService (Web App)
		-> SQL Server -> SQL elastic Pool -> SQL Server -> SQL Database
			(gehört Server zu Pool, oder nimmt Server seine Ressourcen aus einem Pool ?)

RessourceGroup (pro App ?)
	- wir von einer Subscription bezahlt
	- wird als erstes definieren
	- WebApp erstellen
	- Deployment direkt aus VisualStudio möglich
	- Deployment auch aus jedem anderen Tool möglich, da über universelles WebDeploy Protokoll

AppService Plan
	- hat Betriebssystem	
 	- hat pricing tier und steuert somit die Kosten
		- erzeugt fixe Kosten pro Minute, auch ohne Nutzung ! 
	- kann per Scale out auf mehrere Instanzen erhöht werden
	- kann per Scale up auf größeres Pricing Tier erhöht werden
	- B1 Webservice Plan kostet ca. 45 EUR pro Monat fix


AppServices 
	- hosten Web Apps
	- logisch gesehen gemanagter IIS Server
	- incl. Loadbalancer etc.


SQL Server Elastic Database Pool
	- kleinster Eleastic Pool kostet ca. 65 EUR pro Monat (fix, SQL Server Lizenz enthalten)


SQL Server
	- als VM möglich
	- besser als Pool 
	- definiert den admin user 
		- dieser sollte bei Zugriff mit SSMS 2014 keine Email Adresse sein
		- Authenifizierungsmethode Azure AD etc. wird erst ab SSMS 2017 unterstützt

WebAppDeployment
	- Updload ungezippt per FTP
	- Config.ini
	SqlConnectionString[*]=Server=tcp:DataFactory01-sqlserver.database.windows.net,1433;Persist Security Info=False;User ID=support;Password=xxx;MultipleActiveResultSets=False;Encrypt=True;TrustServerCertificate=False;Connection Timeout=30;
	IdentityServerUrl=https://id.planning-factory.com/
	IconServerUrl=https://icons.planning-factory.com/
	WindowsAuthentication=false
	Title=DataFactory

	
Datenbank
	- User müssen per Script angelegt werden
